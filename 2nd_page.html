What are the main components of a robot?

The 5 main components of a robot are the body/frame, the control system, the manipulator, and the drivetrain. The body/frame is the shell of the robot. It surrounds all the other components to make it look pleasing and to protect it. The control system is like the one that does all the processing. The sensors see the robot's surroundings and then give it to the CPU. The CPU then looks at the surroundings and use logic. The manipulators are kind of like your arms. Robots normally need to interact with the things around them so they use manipulators. Unlike the previous two things, robots don't need manipulators. The drivetrains are like your legs. They allow you to move from place to place. Robots that represent humans use legs. However, most robots use wheels to move around.

How are robots programmed?

There are many ways to program a robot but the top three most famous robot teaching methods are the teaching pendant, the simulation/offline programming, and the teaching by demonstration.

The Teaching Pendant

This is most used way to teach robots. The operator has to move the robot and save each position. The robot will memorize each saved position and try to replicate the operator’s movements..The more saved positions, the more accurate the robot will try to replicate the operator’s movements.

The Simulation/Offline Programming

This most used in robotics testing to make sure the algorithm is working correctly before using a real robot.

The Teaching by Demonstration

This my favourite method. A person does a task a bunch of times but with something small different each time like moving the items somewhere else. The robot would see what the person is doing and over time, the robot can do it itself.

What are the advantages and disadvantages of the programming methods?

The Teaching Pendant

Advantages

Most traditional industrial robots come with a teach pendant, which makes them familiar to technicians. The positions are also accurate and are great for simple movements.

Disadvantages

The robot must be put into "teach mode" and all operations using the robot halted until it has been programmed. It requires training to program and might be difficult to skilled craftspeople who are unfamiliar to programming.

The Simulation/Offline Programming

Advantages

The program is developed offline so the robot production is only stopped when the new program is being downloaded and tested and it’s easy to test the different ways to the same problem.

Disadvantages

The virtual models will probably never be the same as in real life, so the program may still need to be altered after the robot is made. It can sometimes end up wasting time sorting out simulator issues instead of solving production challenges.

Teaching by Demonstration

Advantages

You don’t need as many buttons to control the robot. You only need to move it to the desired position and the robot is very easy to code because you need to code it as only a human would perform it. It is also very good for detailed tasks that would normally need many lines of code.

Disadvantages

You need to program the physical robot so it does not reduce as much downtime as offline programming and moving the robot to a precise location is not as easy as the methods. Also, moving a robot for very precise tasks is very hard.
